## Block 3 — Project Governance and Oversight

**Use for:** "Describe your project governance", "How will you manage escalation?", "What oversight controls are in place?"

---

APPSolve applies a three-tier governance model on all engagements:

**Tier 1 — Operational (Weekly)**
- Project status meeting with client PM
- Risk and issue log reviewed and updated
- Progress tracked against agreed milestones (OUM phase gates or Acumatica project plan)
- Action log circulated within 24 hours of each meeting

**Tier 2 — Steering (Monthly)**
- Steering Committee meeting with client Sponsor and APPSolve Engagement Lead
- Budget vs. actuals reviewed
- Change requests formally assessed and approved/deferred
- Key risk escalations presented

**Tier 3 — Executive (Quarterly or at milestone)**
- Executive business review with client CIO/CFO and APPSolve Director
- Strategic alignment review
- Resource adequacy review
- Forward plan for next quarter

**Escalation Path:**
Consultant → Project Manager → Engagement Lead → APPSolve Director (JH Blignaut). Any unresolved matter is escalated to the APPSolve Director within 2 business days of the Engagement Lead being notified.

**Deliverable Management:**
All deliverable documents are subject to formal review and sign-off. APPSolve uses a two-review cycle: internal quality review (QA Lead sign-off before client submission) followed by client acceptance. No configuration change is made to a production environment without a signed Change Request.

## Block 5 — Project Methodology

**Use for:** "What project methodology do you follow?", "Describe your implementation approach", "How do you manage ERP implementations?"

---

**Oracle Engagements — Oracle Unified Method (OUM):**

APPSolve follows the Oracle Unified Method (OUM) for all Oracle ERP, HCM, and Cloud implementations. OUM is Oracle's recommended implementation framework and is structured around five phases:

1. **Inception** — Scope definition; business case validation; project organisation; risk assessment
2. **Elaboration** — Requirements workshops (FRD); process mapping (as-is / to-be); fit-gap analysis; architecture design
3. **Construction** — Configuration; customisation; data migration design; integration development; unit testing
4. **Transition** — System integration testing (SIT); user acceptance testing (UAT); training delivery; cutover planning
5. **Production** — Go-live; hypercare (minimum 4 weeks post-go-live); knowledge transfer; handover to AMS

Each OUM phase gate requires formal deliverable sign-off before the next phase commences. APPSolve does not proceed to Construction without an approved Fit-Gap Analysis signed by the client.

**Acumatica Engagements — Acumatica Implementation Methodology (AIM):**

For Acumatica implementations, APPSolve applies the Acumatica Implementation Methodology (AIM), which is structured around: Discovery → Design → Build → Test → Deploy → Go-Live → Hypercare. The AIM framework is designed for phased delivery and supports modular go-live (e.g., Finance first, Distribution follow-on).

**Agile Elements:**
Both OUM and AIM are applied with Agile elements where appropriate — sprint-based Configuration and Testing cycles, daily standups during SIT/UAT, and iterative demonstration sessions with the business. APPSolve holds certified Agile practitioners (Esna Pretorius — Agile PM, UCT; Michael Palombo — ICAgile) who lead Agile-adapted delivery.

## Block 9 — Cybersecurity and Data Protection

**Use for:** "How do you manage data security?", "What cybersecurity controls are in place?", "POPIA compliance?"

---

APPSolve takes data protection and cybersecurity seriously across all client engagements. Our controls include:

**Data Handling:** Client data used in implementation (for data migration, testing, or configuration) is handled under a signed Data Processing Agreement or equivalent confidentiality clause. Test environments are not connected to production data sources without client consent and formal approval.

**Cybersecurity Capability:** APPSolve maintains a dedicated cybersecurity consultant (CISA-certified) who provides IT audit, penetration testing, cloud security assessment, and ISO 27001 / NIST advisory services. This resource can be deployed to assess and harden client environments as part of or alongside ERP implementations.

**POPIA:** APPSolve is committed to compliance with the Protection of Personal Information Act (POPIA). Consultant access to client personal information is governed by role-based access control and documented in the project security plan at engagement inception.

**Remote Access:** All remote access to client systems is performed via client-approved VPN or secure desktop solutions. APPSolve does not maintain persistent connections to client production environments outside of contracted AMS arrangements.
