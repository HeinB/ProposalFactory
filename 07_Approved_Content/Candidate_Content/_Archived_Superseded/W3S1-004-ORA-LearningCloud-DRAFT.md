## Section 1: Statement of Capability

APPSolve implements Oracle Fusion Learning Cloud as part of Oracle Fusion HCM programmes, enabling organisations to train, develop, and upskill employees through a modern, AI-enabled learning management system natively integrated with Oracle Fusion HCM.

APPSolve has implemented Oracle Fusion Learning Cloud for Mr Price Group, one of South Africa's largest retail groups, and provides ongoing support across the client's broader Oracle HCM landscape. Mr Price Group represents APPSolve's referenceable Oracle Fusion Learning Cloud implementation.

APPSolve's Oracle Learning Cloud delivery experience spans the retail sector. APPSolve supports clients across Retail, Mining, and Professional Services industries.

Compliance Training assignment and Certification Tracking are confirmed APPSolve implementation deliverables, configured as part of every Oracle Fusion Learning Cloud engagement. Oracle Fusion Learning Cloud also includes AI-driven learning recommendations and personalised learning journeys. APPSolve configures and enables these capabilities as part of Oracle Fusion Learning Cloud implementations.

A clear capability boundary applies throughout this document: Oracle Fusion Learning Cloud delivers course management, learning assignments, completion tracking, and certification status monitoring. Advanced learning dashboards and analytics are delivered through Oracle HCM Analytics — a complementary Oracle product. This distinction is maintained throughout Sections 9 and 12.

## Section 2: Oracle Fusion Learning Cloud — Module Overview

Oracle Fusion Learning Cloud is Oracle's next-generation learning management system, forming part of the Oracle Fusion Cloud HCM Suite. APPSolve configures and implements Oracle Fusion Learning Cloud to enable organisations to create, manage, and track learning programmes, ensuring employees have the skills and knowledge required to support business performance.

The platform delivers five primary value outcomes that APPSolve configures as part of every Oracle Fusion Learning Cloud implementation:

| Value Outcome | Description |
|---|---|
| **AI-Powered and Personalised Learning** | Oracle Fusion Learning Cloud uses AI and machine learning to recommend courses based on an employee's role, skills, and career goals. APPSolve configures and enables these capabilities as part of Learning Cloud implementations. |
| **Continuous Learning and Upskilling** | Supports on-the-job training, self-paced courses, instructor-led training (ILT), virtual classrooms, and social learning. Enables employees to stay current with new skills, certifications, and compliance requirements. |
| **Employee Engagement and Knowledge Retention** | Encourages learning through gamification, peer collaboration, and mobile-friendly content. Interactive formats including videos, quizzes, and discussions support knowledge retention. |
| **Compliance and Certification Management** | Automates compliance tracking and certification renewal processes. Ensures employees complete mandatory training and meet regulatory requirements. APPSolve configures Compliance Training and Certification Tracking as standard implementation deliverables. |
| **Seamless Integration with Oracle Fusion HCM** | Natively connects with Oracle Talent Management, Career Development, Performance Management, and Oracle Core HR. Supports integration with third-party content providers and learning platforms where required. |

## Section 3: Learning Administration and Course Catalog Management

### 3.1 Learning Administration Framework

APPSolve establishes the learning administration framework as a core implementation deliverable. This framework defines the structural elements within which client learning administrators manage courses, assign training, and track completion independently after go-live.

| Framework Element | APPSolve Configuration Approach |
|---|---|
| **Learning Administration governance** | APPSolve designs the learning administration structure — courses, offerings, and activities — in collaboration with client HR and L&D teams during the Scope and Design phase |
| **User roles and access** | Learning Administrator, Manager, and Learner roles configured with appropriate access to course management, assignment, and reporting functions |
| **Global learning portal** | One learning portal configured as a global function across all territories within implementation scope |
| **Skills transfer to administrators** | APPSolve trains client learning administrators to own and manage the course catalog, enrolments, and learning assignments post go-live. Client independence is a core delivery objective. |

### 3.2 Course Catalog and Content Management

| Capability | Detail |
|---|---|
| **Centralised course catalog** | APPSolve configures a comprehensive catalog of formal and informal learning content — courses, videos, documents, web links, and structured activities |
| **Supported content formats** | SCORM, AICC, PDF, videos, assessments, and weblinks. All learning content must be provided by the client in one of these Oracle-supported formats |
| **Content authoring** | Content cannot be created directly within Oracle Fusion Learning Cloud. Client content is uploaded to the configured catalog. APPSolve does not produce learning content; content creation is a client responsibility |
| **Catalog population** | Populating the Learning Catalog (Courses, Offerings, and Activities) and learner enrolments must be performed by client learning administrators after go-live. APPSolve configures the catalog structure; the client populates it. |
| **Historical data upload** | Upload of existing completion records is limited to a maximum of 2 years of data for active employees only, in the Oracle-supported import format |

## Section 4: Learning Assignments and Compliance Training

### 4.1 Learning Assignments

APPSolve configures Oracle Fusion Learning Cloud's assignment capabilities to enable structured delivery of mandatory, recommended, and role-based learning across the workforce.

| Assignment Type | Configuration Approach |
|---|---|
| **Mandatory assignments** | Assignments configured as required learning for defined employee groups (by role, department, location, or job grade). Automated enrolment and deadline tracking. |
| **Recommended assignments** | Learning suggested to employees based on role, skills profile, or career development plan. Integrated with Oracle Talent Profile where configured. |
| **Manager-assigned learning** | Managers assign learning to direct reports through the manager self-service portal, with task tracking and completion visibility |
| **Group and population assignments** | Bulk assignment to defined employee populations using Oracle HCM role-based rules and HR hierarchy |

### 4.2 Compliance Training

Compliance Training is a confirmed APPSolve implementation capability, configured as a standard deliverable in Oracle Fusion Learning Cloud implementations.

| Compliance Capability | Description |
|---|---|
| **Mandatory compliance courses** | Configurable mandatory course assignments for regulatory, policy, and safety training requirements. APPSolve configures enforcement rules, deadline windows, and escalation behaviours. |
| **Automated enrolment** | Employees are automatically enrolled in required compliance training based on configured criteria (role, grade, location, regulatory obligation) |
| **Completion tracking** | Course completion status tracked in real time at individual, team, and organisational level. Non-completion alerts generated for managers and administrators |
| **Regulatory training management** | Supports multi-jurisdictional compliance requirements where a single implementation serves employees across different regulatory environments |
| **SETA and ATR reporting** | Legislative reporting (SETA, ATR, WSP) for South African compliance requirements is supported by means of a structured data extract that can be maintained and updated by client HR and skills development administrators |

## Section 6: Self-Service Learning and Mobile Access

### 6.1 Employee Self-Service Learning

APPSolve configures Oracle Fusion Learning Cloud's self-service capabilities to enable employees and managers to manage learning independently without requiring HR administration for routine enrolment and progress tracking.

| Self-Service Capability | Description |
|---|---|
| **Course browsing and enrolment** | Employees browse the learning catalog, enrol in available courses, complete e-learning modules, and withdraw from non-mandatory learning through a self-service portal |
| **Learning progress tracking** | Employees view their own learning history, completion status, outstanding assignments, and certification records through the learner portal |
| **Application tracking** | Employees track the status of learning enrolment requests and approvals in real time |
| **Manager self-service** | Managers view the learning status of direct reports, approve learning enrolment requests, and assign training through the manager self-service portal |
| **Notifications and reminders** | Automated notifications for upcoming deadlines, new assignments, and completion confirmations — configurable at the employee, manager, and administrator level |

### 6.2 Mobile Learning and Offline Access

Oracle Fusion Learning Cloud provides a mobile application that enables employees to access learning content from any device, at any time.

| Mobile Capability | Detail |
|---|---|
| **Mobile application** | Oracle Fusion Cloud Learning supports mobile access via the Oracle HCM mobile app. Employees can browse, enrol, and complete learning on mobile devices. |
| **Offline learning** | Employees can download content (videos, PDFs) to their mobile devices and complete it offline. Content is synchronised upon reconnection. |
| **Offline limitations** | Certain interactive elements — including evaluations, assessments, and some SCORM content — may require online access for submission. Offline capability applies primarily to consumable content. |
| **Responsive design** | Oracle Learning Cloud is accessible through a browser on any device. No separate mobile configuration is required for browser-based access. |

| AI Capability | Description | Framing |

## Section 9: Learning Analytics — Capability Scope and Boundaries

A clear distinction must be maintained between Oracle Fusion Learning Cloud tracking capabilities and Oracle HCM Analytics (OAX) advanced analytics. This distinction is reflected in every APPSolve Learning Cloud delivery and is embedded in the standard implementation assumptions (Section 15).

### 9.1 Oracle Fusion Learning Cloud — Built-In Tracking

The following tracking capabilities are available within Oracle Fusion Learning Cloud as part of the standard implementation scope:

| Tracking Capability | Description |
|---|---|
| **Enrolment tracking** | Real-time visibility of learner enrolment status across all assignments and self-enrolled courses |
| **Completion tracking** | Course completion rates at individual, team, and population level |
| **Assessment results** | Learner assessment scores and outcomes recorded within the learning record |
| **Certification status** | Current, expiring, and lapsed certification status visible at the individual learner and population level |
| **Learner progress** | Stage-by-stage progress through structured learning paths and mandatory compliance programmes |
| **Assignment compliance status** | Percentage completion of mandatory training assignments across defined employee groups — used for regulatory reporting extracts |

**Standard implementation assumption (confirmed across multiple APPSolve delivery engagements):** Oracle Fusion Learning Cloud does not include pre-built Learning Reports or Dashboards as part of the standard implementation scope. The tracking capabilities above are accessible through administrator consoles and learner portals, not through formal report interfaces.

### 9.2 Oracle HCM Analytics — Advanced Learning Analytics

Advanced learning analytics — including executive dashboards, trend analysis, skills intelligence, and cross-module reporting — are delivered through Oracle HCM Analytics (OAX), which is a separately licensed and separately implemented Oracle product.

| OAX Learning Analytics Capability | Description |
|---|---|
| **Learning dashboards** | Pre-built OAX dashboards covering learning completion rates, compliance metrics, skills development progress, and training ROI |
| **Skills intelligence** | Advanced analytics connecting learning activity to skills attainment across the workforce |
| **Trend analysis** | Historical and predictive analysis of learning engagement, completion patterns, and skill gap closure over time |
| **Executive analytics** | Senior leadership reporting on workforce learning investment and capability development outcomes |
| **Cross-module analytics** | Connecting learning data with performance, talent, and workforce composition data for holistic people analytics |

Clients requiring advanced learning dashboards and analytics should reference the APPSolve Oracle HCM Analytics and Workforce Intelligence capability statement (W3S1-006). APPSolve supports end-to-end Oracle HCM Analytics implementations where advanced learning reporting is a delivery requirement.

*Approved positioning: "Advanced learning analytics and dashboarding capabilities are typically delivered through Oracle HCM Analytics rather than Oracle Fusion Learning Cloud itself."*

## Section 12: Integration Architecture

Oracle Integration Cloud (OIC) is APPSolve's standard integration layer in every Oracle Fusion HCM implementation, including Oracle Fusion Learning Cloud. This applies without exception.

### 12.1 Standard Learning Cloud Integration Points

| Integration | Method | Description |
|---|---|---|
| **Oracle Fusion Core HR** | Native Oracle integration | Learning assignments and completion records linked to employee records in Core HR. Learner populations defined by HR hierarchy, role, and grade from Core HR. |
| **Oracle Talent Management** | Native Oracle integration | Learning outcomes recorded in Oracle Fusion Talent Profile. Skill gap identification connects Talent Management to Learning Cloud recommendations. |
| **Oracle Performance Management** | Native Oracle integration | Performance review outcomes can trigger learning recommendations. Development plan objectives linked to learning path assignments. |
| **Oracle Career Development** | Native Oracle integration | Career goals in Oracle Career Development drive learning path recommendations. Learning completion supports career readiness tracking. |
| **Oracle Onboarding** | Native Oracle integration | Learning assignments for new hires integrated with Oracle Onboarding workflows. Mandatory onboarding training tracked through Oracle Learning Cloud where configured. |
| **SETA / WSP / ATR reporting** | OIC / HCM Extracts | South African skills development legislative reporting delivered through structured data extracts. APPSolve configures the extract mechanism; client administrators manage ongoing reporting. |
| **Third-party content providers** | OIC / REST APIs | APPSolve can configure Oracle Fusion Learning Cloud integrations with supported third-party learning content providers and learning platforms (such as LinkedIn Learning, Coursera, and equivalent platforms) where required. Scope and provider-specific integration requirements confirmed during Scope and Design phase. |
| **External LMS platforms** | REST APIs / OIC | Where clients operate an existing LMS (e.g., Moodle) alongside Oracle Learning Cloud, APPSolve can configure integration to align completion records, reporting extracts, and user data between platforms. |

### 12.2 REST API and Data Loader Support

Oracle Fusion HCM provides comprehensive RESTful APIs and HCM Data Loader (HDL) capabilities for learning administration:

- **HDL templates** — APPSolve configures HDL templates for bulk loading and editing of learning records (course completions, certifications, enrolments)
- **HCM Spreadsheet Data Loader (HSDL)** — supports mass import of learning assignments and historical completion records
- **Learning reports export** — learning activity data can be exported for offline analysis and legislative reporting

## Section 13: Implementation Approach

### 13.1 Learning Cloud within the Oracle Fusion HCM Programme

Oracle Fusion Learning Cloud is implemented as a structured delivery phase within the broader Oracle Fusion HCM programme. Learning typically follows Core HCM and Recruiting in the implementation sequence, ensuring that the employee population and job profile structure are in place before Learning configuration begins.

| Phase | Content | Learning Cloud Dependency |
|---|---|---|
| Phase 1 | Core HCM — Global HR, Absence Management, Self-Service | Foundation — employee records, job structure, HR hierarchy required by Learning |
| Phase 2 | Talent Management / Recruiting | Talent Profile and competency framework required for skills-driven learning recommendations |
| **Phase 3** | **Oracle Fusion Learning Cloud** | **Requires Phase 1 (employee population, roles); Phase 2 (Talent Profile, competency framework) for full AI recommendation capability** |
| Phase 4 | Workforce Compensation, Help Desk | Parallel or subsequent to Learning |

### 13.2 OUM Delivery Phases — Learning Cloud

APPSolve applies the Oracle Unified Methodology (OUM) across all Oracle Fusion HCM implementations, including Learning Cloud:

| OUM Phase | Learning Cloud Activities |
|---|---|
| **Mobilize** | Project charter; Learning Cloud scope confirmation; framework design workbooks; stakeholder alignment on content ownership and compliance requirements |
| **Scope and Design** | Learning administration framework design; compliance training inventory; certification requirements mapping; SETA/WSP/ATR extract design; content format review; integration architecture confirmation |
| **Prototype** | CRP1 — learning administration framework walk-through; compliance assignment testing; CRP2 — refined configuration including catalog structure, assignment rules, certification controls, and mobile access |
| **Build** | Full configuration build; HDL templates for historical data load; OIC integration build (content providers, SETA extract, external LMS where in scope); training material development; skills transfer sessions with learning administrators |
| **Deploy** | Production build; data verification (historical completions — up to 2 years, active employees only); UAT; administrator training; go-live support |
| **Post Go-Live** | Hyper-care (full team approximately 3 months); catalog population support; administrator independence confirmation; steady-state handover |
