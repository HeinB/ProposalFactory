### 4.3 Assumption ID Format Variants

The extraction regex handles six distinct body formats across all 13 packs:

| Format | Pattern | Example Pack |
|---|---|---|
| Multiline | `**ID**\ntext on next line` | HCM, ERP, OIC, OCI, AMS |
| Inline-period | `**ID.** text continues` | OCI, OIC |
| Inline-colon | `**ID:** text continues` | BeBanking |
| Bracket-annotation | `**ID [REPLACES: X]**` | AMS overlays |
| Em-dash label | `**ID — Label**` (stripped to `ID`) | AMS SLA overlay |
| Digit-category | `**PREFIX-3PT-NNN**` | HCM base |

All six formats extract correctly. IDs with annotations are stored in canonical form (annotations stripped from `asset_id`; full raw form preserved in source document).
