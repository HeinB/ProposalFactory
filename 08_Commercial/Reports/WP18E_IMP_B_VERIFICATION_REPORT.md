## 4. Defects Found and Resolved

### Pre-Fix State (Build 3)

| Defect | Severity | Affected Assets |
|---|---|---|
| D-IMP-001: ASM IDs contained em-dash annotations | **BLOCKING** | 6 EBS SLA assumptions |
| D-IMP-002: W5-METH-001 empty owner_business_unit | WARNING | 1 CAP |
| D-IMP-003: 6 Corporate CAPs had cap_ext.platform = Cross_BU | WARNING | 6 CAPs |
| D-IMP-004: 31 Wave 1/2 CAPs had empty `created` field | WARNING | 31 CAPs |

### krpe.py Fixes Applied

**D-IMP-001 fix** — `clean_id` stripping extended to cover em-dash labels:
```python
clean_id = re.sub(r"\s+(?:\[.*?\]|[—–]\s*.+)$", "", raw_id).strip()
```

**D-IMP-002 fix** — `_derive_bu()` extended to check `bu:` field for Wave 5 files:
```python
if not bu:
    bu = _str(file_meta.get("bu", ""))
    if bu:
        return _bu_from_raw(bu)
```

**D-IMP-003 fix** — `_BU_NORMALISE` applied to platform derivation:
```python
platform = _BU_NORMALISE.get(raw_platform, raw_platform) if raw_platform in _BU_NORMALISE else owner_bu
```

**D-IMP-004 fix** — Wave 1/2 field aliases added to all date/author fields:
```python
"created": _str(file_meta.get("created", "") or file_meta.get("extraction_date", "")),
"created_by": _str(file_meta.get("created_by", "") or file_meta.get("extracted_by", "")),
"approved_by": _str(file_meta.get("approved_by", "") or file_meta.get("reviewed_by", "")),
```
