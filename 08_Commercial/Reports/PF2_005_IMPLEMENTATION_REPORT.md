## 2. Investigation Findings

### 2.1 Why Assumption Packs Were NOT_FOUND

The renderer's `AssetIndex` class scanned only `07_Approved_Content/Approved/`. ASP files live in
`08_Commercial/Assumptions/`. Additionally, ASP filenames use underscores (`OIC_ASSUMPTIONS_V1.md`)
while canonical IDs use hyphens (`OIC-ASSUMPTIONS-V1`). The existing `_short_id()` extraction logic
could not bridge this gap since ASP IDs contain no all-digit segment.

### 2.2 No New Files Required

All 13 ASP files already existed as approved governed content. The solution was purely an indexing
extension — no new Markdown files were authored, and no content was duplicated.

---

## 3. Implementation

### 3.1 BeBanking document_id Fix

`08_Commercial/Assumptions/BeBanking/BEBANKING_BASE_ASSUMPTIONS_V1.md` was missing a `document_id`
field. Added `document_id: BEBANKING-BASE-ASSUMPTIONS-V1` to YAML frontmatter. This was a one-line
governance defect fix; all other 12 ASP files were already compliant.

### 3.2 AssetIndex Extension

**File:** `08_Commercial/Assembly_Engine/proposal_renderer.py`

Changes made:
1. Added `ASSUMPTIONS_ROOT` constant pointing to `08_Commercial/Assumptions/`
2. Extended `AssetIndex.__init__` to accept `additional_roots: Optional[List[str]]`
3. Added `AssetIndex._extract_document_id(path)` static method — reads YAML frontmatter,
   returns `document_id` value or `None`
4. Extended `AssetIndex._build(root, index_by_document_id)` — when `index_by_document_id=True`,
   indexes files by `document_id` from frontmatter rather than filename stem
5. Updated CLI instantiation: `AssetIndex(args.content_root, additional_roots=[ASSUMPTIONS_ROOT])`

### 3.3 Design Principles Preserved

- **Single source of truth:** ASP files remain in `08_Commercial/Assumptions/`; no copies created
- **document_id authority:** Frontmatter `document_id` is the canonical identity for ASP lookup
- **No bypass of PSAE:** Manifests are regenerated by PSAE; renderer consumes manifest as before
- **No registry modification:** Existing 13 ASP registry entries unchanged
- **Backward compatibility:** Primary root scanning unchanged; additional_roots is additive
