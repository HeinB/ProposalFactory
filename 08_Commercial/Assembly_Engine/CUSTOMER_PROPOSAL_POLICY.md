# Customer Proposal Policy v1.0

## What a Customer Proposal Must Contain

| Section | Content Standard |
|---|---|
| Executive Summary | Win theme, APPSolve positioning, key differentiators, call to action |
| Understanding of Requirements | Client-specific context, objectives, pain points — NOT generic |
| Proposed Solution Overview | Confirmed scope (modules, entities, phasing) |
| Company Credentials | APPSolve overview, awards, delivery model, geography |
| Implementation Methodology | Oracle Modern Best Practices narrative — NO OUM |
| Oracle HCM Capability | Business-outcome capability per module — budget-capped |
| Project Governance | Governance table, decision-making framework |
| Team Structure | Named consultants or roles (depending on client preference) |
| Key Assumptions | Executive summaries (3–8 bullets per pack) |
| Risk Register | Material risks with mitigations |
| Commercials / Pricing | Placeholder for commercial proposal — no pricing in this document |
| Client References | Anonymous summaries — AM approval required for named references |
| Compliance Documents | Index of documents to be provided |
| Assumption Schedule (Appendix) | Executive summaries per pack — NOT full libraries |

---

## What a Customer Proposal Must NOT Contain

| Item | Policy |
|---|---|
| Oracle Unified Method (OUM) | R6: Prohibited for Fusion Cloud — replace with Oracle Modern Best Practices |
| Internal asset IDs (CAP-..., ASM-...) | R2: Always removed |
| BU decision tables | R4: Always removed |
| Individual assumptions with IDs | R5: Replace with executive summaries |
| Source evidence / primary source columns | R9: Always removed |
| Approval records / governance review notes | R1: Always removed |
| Extraction notes, fact verification | R1: Always removed |
| Implementation annotation history | R3: Always removed |
| Pricing, rates, or commercial estimates | Separate Commercial Proposal required |
| Named references without AM approval | Governance constraint |
| B-BBEE status after 2026-07-31 | Governance constraint — renewal required |
| Oracle Gold Partner claim | Expired August 2021 — prohibited |

## Target Size

| Metric | Target |
|---|---|
| Total pages | 25–35 |
| Maximum pages | 40 |
| Assumption schedule | ≤ 6 pages (executive summaries only) |
| Capability sections total | ≤ 15 pages |
| Company and credentials | ≤ 5 pages |

---

## Quality Gates Before Customer Release

| Gate | Check |
|---|---|
| Q1 | No OUM reference in any section |
| Q2 | No internal IDs visible in body text |
| Q3 | Executive Summary is tender-specific (not generic) |
| Q4 | Assumption sections are executive summaries (not assumption libraries) |
| Q5 | No BU decision tables visible |
| Q6 | No source evidence columns in any table |
| Q7 | Compliance documents: B-BBEE certificate is current (not after 2026-07-31) |
| Q8 | References approved by AM before naming any client |
| Q9 | Commercial Proposal issued separately — no pricing in this document |
| Q10 | Total size ≤ 40 pages |

All gates must pass before customer release.
